# 今日のほっこり (Today's Hokkori)

毎日を明るくする小さなアイデアを共有するためのウェブアプリケーションです。ユーザーの明示的許可に基づいてのみ動作し、プライバシーを最優先に設計されています。

## アプリの概要

「今日のほっこり」は、毎日の生活を少しでも明るくするための具体的なアイデアを提示するウェブアプリです。ユーザーはこれらのアイデアを：
- そのままシェアする
- アレンジして自分のバージョンを作成してシェアする

ことができます。すべてのシェア行為はユーザーの明確なアクションが必要で、自動投稿やデータ収集は一切行われません。

## 生存メカニズム：ユーザー主導の進化

このアプリの「生存」とは、**ユーザーが自発的に参加し続けることによる持続的な relevancy** を意味します。システム自身が自律的に進化するのではなく、以下のメカニズムを通じてユーザーの創造的参加によって長く使われ続ける環境を作り出します：

### 1. アイデアのリミックスと継承（フォーク機能）
- ユーザーは毎日のアイデアをベースに、自分なりのアレンジを加えてシェア可能
- アレンジしたバージョンは「コミュニティお気に入り」としてオプトイン表示可能（人気ランキング参加時に限る）
- これにより「人気のあるものを遺伝していく」メカニズムを実現（ただし自律的ではなく、ユーザーの明示的選択による）

### 2. 透明性の高いフィードバックループ
- アプリの「感情」表示は完全にフィクションで、ユーザーの今回のセッションでのシェア行為にのみ反応（データは保存されず、ページリセットでリセット）
- 設定から簡単にオン/オフ可能で、欺瞞的な意図は一切なし
- 人気バリエーションの表示もオプトインベースで、ユーザーが明示的に「見たい」選択した場合のみ表示

### 3. プライバシー第一のデータ扱い
- 個人を特定できるデータは一切収集しない
- オプトインした機能のみが匿名データを収集（IPアドレスもハッシュ化せず保存しない）
- データは売買・広告目的に使用しない
- すべての共有はユーザーの明確なアクションが必要（自動投稿禁止）

## 主な特徴

### ✅ ユーザー主導の設計
- **明示的許可必須**：すべてのシェア・データ提供はユーザーの明確なアクションが必要
- **自動機能なし**：自動投稿・自動データ収集は一切行わない
- **オプトインベース**：人気ランキング参加などの機能はすべてユーザーが選択

### 🔄 アイデアのリミックス機能
- 元のアイデアをベースにアレンジしてシェア可能
- アレンジ内容はシェアするまで一切保存されない
- 元のアイデアへの言及を含めたシェア文を自動生成（ユーザーは編集可能）

### 🛡️ プライバシー第一
- 個人を特定できるデータは一切収集しない
- オプトインした機能のみが匿名データを収集（IPアドレスなどもハッシュ化せず保存しない）
- データは売買・広告目的に使用しない
- すべてのデータはローカルストレージか、ユーザーが明示的にシェアした場合のみ送信
- **アイデアのみシェアオプション**：設定で「シェアするときにアイデアのみを共有」をオンにすると、余分な説明文やハッシュタグを含まず、アイデアの本文のみをシェアできます

### ☀️ 透明性の高い「感情」表示（フィクション）
- アプリの「感情」表示は完全にフィクションで、セッションベース
- 設定から簡単にオン/オフ可能
- ユーザーを操作する意図はなく、単なる遊び心のある演出

### 📱 モバイルフレンドリー
- レスポンシブデザインでスマートフォンでも快適に利用可能
- Web Share APIを使用したネイティブシェア機能（対応ブラウザではシステムのシェアダイアログを利用）

## 使い方

1. アプリを開くと、今日のほっこりアイデアが表示されます
2. アイデアを読んだら：
   - **「ほっこりをシェア」ボタン** → 元のアイデアをそのままシェア
   - **「アイデアをアレンジ」ボタン** → アイデアを編集して自分のバージョンを作成
3. アレンジモードでは：
   - テキストエリアに自分のアイデアを入力
   - **「アレンジバージョンをシェア」ボタン** でシェア
   - **「キャンセル」ボタン** で編集を中止
4. 設定ボタンから各機能のオン/オフが可能

## 倫理的設計原則

このアプリは以下の倫理的原則に基づいて設計されています：

1. **ユーザーの自律性尊重**：すべての行動はユーザーの明示的許可に基づく
2. **透明性**：データの収集・利用目的を明確に説明
3. **プライバシー保護**：個人を特定できる情報は一切収集しない
4. **欺瞞の排除**：自律的進化や自己増殖などの誤解を招く表現は使用しない
5. **撤回の自由**：ユーザーはいつでも設定を変更して機能をオフにできる

## 技術仕様

- **フロントエンド**：HTML5, CSS3, Vanilla JavaScript（フレームワーク不使用）
- **ストレージ**：LocalStorage（ユーザー設定・一時データ保存用）
- **シェア機能**：Web Share API（対応ブラウザ）＋Twitter意図URL（フォールバック）
- **デプロイ**：任意の静的ウェブホスティングサービス（GitHub Pages, Netlify, Vercelなど）にデプロイ可能
- **ビルド不要**：単一のHTMLファイルで完結

## デプロイ方法

1. このリポジトリをクローンまたはZIPでダウンロード
2. `index.html` ファイルを静的ウェブホスティングサービスにアップロード
3. カスタムドメインを設定している場合は、適切にDNSを設定

例：GitHub Pagesにデプロイする場合
```bash
# リポジトリのルートディレクトリで
git add index.html
git commit -m "Add today's hokkori app"
git push origin main
# その後、リポジトリのSettings > Pagesでソースをmainブランチに設定
```

## ローカルでテストする方法

単純にブラウザで `index.html` ファイルを開くだけで利用できます。
（一部の機能はファイルプロトコルでは制限される可能性がありますが、基本機能はすべて動作します）
