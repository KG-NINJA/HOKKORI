# 今日のほっこりアプリ - 実装済み機能サマリー

## コア機能
1. **毎日更新のほっこりアイデア提示**
   - 日付ベースでアイデアをローテーション
   - 具体的で実行可能な小さな行動アイデアを提供

2. **許可ベースの共有システム**
   - すべての共有はユーザーの明示的アクションが必要
   - 自動投稿・自動データ収集は一切なし
   - Web Share APIを使用したネイティブシェア（対応ブラウザ）
   - フォールバック：Twitter意図URL

3. **アイデアリミックス機能**
   - 元のアイデアをベースにアレンジしてシェア可能
   - アレンジ内容はシェアするまで一切保存されない
   - 元のアイデアへの言及を含めたシェア文を自動生成

## 生存メカニズム（ユーザー主導の進化モデル）
1. **ユーザー主導のアイデア進化**
   - ユーザーがアイデアをアレンジしてシェアすることでバリエーションが生成
   - 人気のバリエーションはオプトインで表示可能
   - 「人気のあるものを遺伝していく」メカニズムを実現（ただし自律的ではなく、ユーザーの明示的選択による）

2. **透明性の高いフィードバック**
   - アプリの「感情」表示は完全にフィクションでセッションベース
   - 設定から簡単にオン/オフ可能
   - 欺瞞的な意図は一切なし

## プライバシー保護機能
1. **ゼロデータ収集初期状態**
   - 個人を特定できるデータは一切収集しない
   - IPアドレスもハッシュ化せず保存しない

2. **オプトイン型データ提供**
   - 人気ランキング参加などの機能はすべてユーザーが明示的に選択
   - 収集した匿名データはサービス改善のみに使用
   - データは売買・広告目的に使用しない

3. **ローカルファーストアーキテクチャ**
   - すべてのデータはローカルストレージに保存（設定・一時データ）
   - ユーザーが明示的にシェアした場合のみデータが外部に送信

## 実装済み特別機能
1. **カスタムシェアURL**
   - すべてのシェアに `https://kg-ninja.github.io/HOKKORI/` が含まれる

2. **アイデアのみシェアオプション**
   - 設定で「シェアするときにアイデアのみを共有」をオンにすると
   - 余分な説明文やハッシュタグを含まず、アイデアの本文のみをシェア可能
   - 元のアイデア共有時：アイデアテキストのみ
   - リミックスアイデア共有時：アレンジしたアイデアテキストのみ

## 倫理的設計原則
1. **ユーザーの自律性尊重**：すべての行動はユーザーの明示的許可に基づく
2. **透明性**：データの収集・利用目的を明確に説明
3. **プライバシー保護**：個人を特定できる情報は一切収集しない
4. **欺瞞の排除**：自律的進化や自己増殖などの誤解を招く表現は使用しない
5. **撤回の自由**：ユーザーはいつでも設定を変更して機能をオフにできる

## 技術仕様
- **フロントエンド**：HTML5, CSS3, Vanilla JavaScript（フレームワーク不使用）
- **ストレージ**：LocalStorage（ユーザー設定・一時データ保存用）
- **シェア機能**：Web Share API（対応ブラウザ）＋Twitter意図URL（フォールバック）
- **デプロイ**：任意の静的ウェブホスティングサービス（GitHub Pages, Netlify, Vercelなど）にデプロイ可能
- **ビルド不要**：単一のHTMLファイルで完結（index.htmlのみ）

## 使い方
1. ブラウザで index.html を開く
2. 今日のほっこりアイデアを読む
3. 共有したい場合：
   - 「ほっこりをシェア」ボタン → 元のアイデアを共有
   - 「アイデアをアレンジ」ボタン → アレンジしてから共有
4. 設定で各機能のオン/オフが可能
   - 人気ランキング参加（匿名データ提供）
   - 人気バリエーションを表示
   - アイデアの系譜を表示
   - アプリの「感情」表示（フィクション）
   - シェアするときにアイデアのみを共有 ← 新機能！

## ローカルテスト方法
単純にブラウザで index.html ファイルを開くだけで利用できます。
（一部の機能はファイルプロトコルでは制限される可能性がありますが、基本機能はすべて動作します）

## デプロイ方法
1. このディレクトリのすべてのファイルを静的ウェブホスティングサービスにアップロード
2. カスタムドメインを設定している場合は、適切にDNSを設定

例：GitHub Pagesにデプロイする場合
```bash
# リポジトリのルートディレクトリで
git add .
git commit -m "Deploy today's hokkori app"
git push origin main
# その後、リポジトリのSettings > Pagesでソースをmainブランチに設定
```
